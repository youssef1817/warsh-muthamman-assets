# دليل بيانات مواضع الآيات (طبعة ورش المثمنة)

يحتوي هذا المجلد على البيانات والإحداثيات اللازمة لتظليل الآيات ورسم دوائر نهاية الآيات وعناوين السور في تطبيق القرآن الكريم. 

البيانات الأساسية محفوظة في ملفات `JSON` (ملف لكل صفحة)، ويتم تجميعها في قاعدة بيانات `SQLite` ليستخدمها التطبيق الفعلي.

---

## 🚀 دليل البدء السريع (للمطورين الجدد على ويندوز)

إذا قمت للتو بتحميل المستودع وتريد البدء في تعديل أو مراجعة مواضع الآيات، اتبع هذه الخطوات البسيطة:

### 1. تشغيل أداة التعديل المرئية (Visual Editor)
لا تقم بتعديل ملفات الـ `JSON` يدوياً! لقد وفرنا أداة مرئية متقدمة تعمل من خلال المتصفح:
1. افتح موجه الأوامر (Terminal) وانتقل لمجلد `warsh-muthamma-assets`.
2. قم بتشغيل الخادم المحلي:
   ```cmd
   node viewer/server.js
   ```
3. افتح متصفحك على الرابط: `http://localhost:3000`
4. من هناك يمكنك تعديل إحداثيات الآيات، سحب المربعات، وحفظ التغييرات بسهولة عبر واجهة رسومية وميزات اختصارات متقدمة.

### 2. مزامنة التعديلات مع تطبيق الموبايل (لتجربة صفحة واحدة)
عندما تقوم بتعديل صفحة في الأداة (مثلاً الصفحة رقم 26) وتريد رؤية النتيجة فوراً داخل التطبيق الحقيقي (على الويندوز أو محاكي الأندرويد):
1. افتح نافذة Terminal جديدة.
2. شغل سكريبت المزامنة السريع مع تمرير رقم الصفحة:
   ```cmd
   tools\sync_warsh_muthamma_ayahinfo.cmd -26
   ```
3. سيقوم السكريبت بحقن التعديلات في قاعدة البيانات الأصلية، ثم ينسخها تلقائياً لمسارات عمل التطبيق في حاسوبك والمحاكي.
4. قم بعمل `Hot Reload` أو `Hot Restart` لتطبيق Flutter لتظهر التعديلات.

### 3. إعادة بناء قاعدة البيانات بالكامل (Rebuild)
إذا قمت بتعديلات كثيرة جداً أو أردت بناء قاعدة البيانات `quran.ar.warsh_muthamma.db` بالكامل من الصفر بناءً على كافة ملفات الـ `JSON` النهائية، قم بتشغيل:
```cmd
node tools\06_rebuild_ayahinfo.js
```
هذا الأمر سيقوم بتجميع كافة الإحداثيات الـ 485 وبناء الجداول من جديد لتكون جاهزة للإطلاق.

## 📂 أين تُحفظ البيانات؟
- **`pages_json/`**: تُحفظ هنا ملفات إحداثيات الآيات ونهاياتها (التعديل اليومي يكون هنا عبر الأداة).
- **`page_layout_json/`**: تُحفظ هنا إعدادات هيكل الصفحة (حدود الأسطر العمودية، البسملة). 
- **`quran.ar.warsh_muthamma.db`**: قاعدة البيانات الجاهزة التي يقرؤها التطبيق النهائي.
